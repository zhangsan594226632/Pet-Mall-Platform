## 一、项目概述

### 1\.1 项目背景

随着养宠人群规模持续扩大，线下宠物店存在品类有限、价格不透明、选购不便、缺少线上交流渠道等短板，传统线下交易模式无法满足用户线上浏览、选购、交流宠物、查询养护资讯的需求。为打通线上宠物交易服务链路，搭建集宠物展示、线上购买、资讯公告、互动论坛、订单管理于一体的数字化平台，开发**基于 SpringBoot\+Vue 宠物商城平台**。

系统采用前后端分离架构，分为**用户前台商城端**与**管理员后台管理端**，普通用户可在线浏览宠物、下单购买、查看行业资讯、论坛互动；管理员统一管控宠物分类、宠物商品、用户、订单、公告、论坛数据，实现宠物交易规范化、线上化管理。

### 1\.2 开发目标

1. 双端权限隔离：前台面向普通消费者实现宠物选购、资讯浏览、互动留言、下单支付；后台由管理员完成全平台数据运维管控。

2. 前台用户核心功能：账号注册 / 登录、首页宠物推荐、宠物分类筛选、宠物详情查看、收藏点赞、加入购物车、立即购买、通知公告浏览、论坛交流、个人订单管理、个人信息维护。

3. 管理员后台核心功能：可视化数据大屏、用户管理、宠物分类管理、宠物信息管理、通知公告管理、论坛交流管理、订单管理、系统配置、个人资料修改。

4. 通用基础功能：多条件模糊检索、分页展示、新增 / 编辑 / 删除批量操作、图片上传、点赞收藏、评论管理、表单校验、角色权限区分。

5. 完整业务闭环：管理员录入宠物分类与宠物商品→前台用户浏览筛选、收藏咨询、下单购买→后台统一管理订单、处理用户评论、发布行业养护资讯、维护论坛互动内容。

### 1\.3 系统角色划分

|角色|权限范围|
|---|---|
|普通用户（前台）|注册登录、浏览全部宠物、按分类筛选、查看宠物详情、点赞 / 收藏宠物、加入购物车、下单购买、查看通知公告、论坛发帖 / 评论、管理个人订单、修改账号信息|
|系统管理员（后台）|可视化数据统计、管理平台全部用户、维护宠物分类、新增 / 编辑 / 下架宠物商品、发布 / 管理养护公告、审核论坛帖子、处理全平台订单、系统基础配置、后台个人资料修改|

## 二、整体技术架构

### 2\.1 架构模式

采用标准**前后端分离架构**，页面展示与业务逻辑完全解耦：

1. 后端服务：SpringBoot 搭建 RESTful 接口，统一处理业务逻辑、事务、权限校验、数据库交互；

2. 用户前台：Vue3 \+ Element Plus 开发商城展示页面，面向购宠用户；

3. 管理后台：Vue3 \+ Element Plus 轻量化管理系统，用于平台运维；

4. 数据存储：MySQL8\.0 持久化用户、宠物、订单、公告、论坛全量业务数据；

5. 身份认证：JWT 无状态令牌，区分管理员 / 普通用户两种角色，拦截越权接口访问。

### 2\.2 后端技术栈

- 核心框架：SpringBoot 2\.7\.x，内置 Tomcat，简化配置、一键启动；

- Web 层：SpringMVC，负责请求分发、参数统一校验、结果封装返回；

- 持久层：MyBatis \+ MyBatis\-Plus，封装通用 CRUD、分页、逻辑删除，简化数据库操作；

- 数据库：MySQL 8\.0，事务保障下单、库存变更、订单生成等核心数据一致性；

- 安全认证：JWT \+ 全局拦截器，实现登录鉴权、角色权限控制；

- 工具类：Lombok 简化实体类、Hutool 实现图片上传、Validation 表单校验、SLF4J 日志记录；

- 分层规范：Controller 控制层、Service 业务层、Mapper 持久层、Entity 实体、DTO 传输对象、Config 配置、Common 工具包。

### 2\.3 前端技术栈（前台 \+ 后台统一）

- 核心框架：Vue3 组合式 API \+ Vite 快速构建项目；

- 路由：Vue Router，路由守卫拦截未登录用户访问购物车、订单、个人中心；

- 状态管理：Pinia，存储登录账号、角色、购物车临时数据；

- UI 组件：Element Plus，表格、弹窗、图片上传、分页、搜索、数据图表等通用组件；

- 网络请求：Axios 统一封装请求拦截器，自动携带 Token、捕获异常弹窗提示；

- 可视化图表：ECharts，实现后台用户性别、宠物分类、库存、品牌数据统计；

- 样式：SCSS \+ Flex 弹性布局，适配 PC 端浏览器。

### 2\.4 开发与部署工具

- 后端开发：IDEA、Maven3\.6、Navicat、Postman 接口调试；

- 前端开发：VS Code、Node\.js 16、NPM；

- 版本管理：Git；

- 部署方式：后端 Jar 包独立运行，Nginx 反向代理前端静态资源，支持 Windows、Linux 服务器部署。

### 3\.1 前台用户商城模块

#### 3\.1\.1 登录注册模块

独立登录页面，提供账号密码登录；新用户可跳转注册页，填写账号、密码、姓名、年龄、性别并上传头像完成注册；未登录状态无法使用购物、收藏、发帖功能。

#### 3\.1\.2 商城首页模块

轮播广告 banner 展示平台优惠活动；Best sellers 热门宠物推荐卡片展示在售宠物缩略图、名称、点赞、收藏、评论数量；提供「查看更多」入口跳转宠物列表页。

#### 3\.1\.3 宠物信息模块

1. 宠物列表：按宠物分类标签快速筛选，支持名称、价格区间多条件搜索；卡片展示宠物配图、名称、基础互动数据；

2. 宠物详情页：展示宠物多图预览、价格、限购数量、宠物分类、年龄、性别、配送方式、资质、规格、上架时间、点击量；提供收藏、点赞、踩一踩功能；支持调整购买数量，添加购物车 / 立即购买；下方分商品介绍、评论标签页，查看宠物详情描述与用户真实评论。

#### 3\.1\.4 通知公告模块

平台宠物养护、活动公告展示页面，支持按标题、简介模糊检索；卡片展示公告封面、标题、发布时间、点赞收藏数据，侧边栏展示热门公告推荐。

#### 3\.1\.5 购物车模块

存放用户意向宠物，支持修改购买数量、批量勾选、结算下单；下单时校验库存、生成订单。

#### 3\.1\.6 论坛交流模块

用户可发布养宠心得、求助帖子，浏览他人帖子并评论互动，形成线上养宠交流社区。

#### 3\.1\.7 个人中心模块

- 账号资料：修改昵称、头像、登录密码；

- 我的收藏：查看所有收藏的宠物；

- 我的订单：全部待支付、已支付、已完成订单，查看订单详情；

- 我的帖子 / 评论：管理本人发布的论坛内容。

### 3\.2 管理员后台管理模块

左侧固定侧边导航，包含返回主页、用户、宠物分类、宠物信息、通知公告、论坛交流、系统管理、订单管理、个人资料九大功能菜单。

#### 3\.2\.1 后台登录与数据首页

管理员专属登录入口；登录后进入可视化数据大屏，包含四大 ECharts 图表：

1. 用户人数饼图：统计平台男女用户占比；

2. 宠物分类环形图：展示各品类宠物商品数量分布；

3. 商品库存柱状图：展示各分类宠物库存数据；

4. 品牌比例折线图：统计各宠物品牌商品数量走势。

#### 3\.2\.2 用户管理

管理平台全部注册消费者账号，支持多条件检索、新增、编辑、禁用、删除用户，查看用户下单、收藏、发帖记录。

#### 3\.2\.3 宠物分类管理

维护宠物一级分类数据，列表展示分类名称；提供搜索、新增、批量移除、单条阅读 / 修改 / 删除操作，分类数据同步至前台筛选栏。

#### 3\.2\.4 宠物信息管理

全量宠物商品数据列表，展示名称、所属分类、规格、上架日期、配图、库存、售价、点赞数；支持阅读详情、修改宠物信息、查看用户评论、移除下架宠物；分页展示，支持多条件搜索筛选。

#### 3\.2\.5 通知公告管理

#### 3\.2\.6 论坛交流管理

#### 3\.2\.7 订单管理

#### 3\.2\.8 系统管理

#### 3\.2\.9 个人资料

## 五、核心业务流程

### 5\.1 宠物商品上架与浏览流程

1. 管理员后台维护宠物分类，新增对应品类；

2. 进入宠物信息管理，填写宠物名称、规格、价格、图文详情，上传图片后新增宠物商品；

3. 商品自动同步前台宠物列表页面，用户可按分类、价格筛选；

4. 用户点击进入详情页，查看介绍与评论，可点赞、收藏、加入购物车。

### 5\.2 线上购买下单流程

1. 用户浏览宠物，选择数量添加购物车或直接立即购买；

2. 确认收货信息、核对商品与金额，提交订单；

3. 校验账号库存，完成支付后订单状态变更为「已支付」；

4. 管理员在后台订单管理查看待发货订单，确认发货；

5. 用户个人中心查看物流、完成收货。

### 5\.3 公告发布与资讯浏览流程

1. 管理员后台新增通知公告，填写标题、简介、正文并上传封面；

2. 公告同步至前台公告页面，支持检索查看详情；

3. 用户浏览公告可点赞收藏，获取宠物养护、平台活动资讯。

### 5\.4 论坛互动闭环流程

1. 登录用户前台发布养宠帖子，上传配图；

2. 所有用户可浏览帖子并发表评论；

3. 管理员后台审核全部帖子，删除违规内容，维护社区环境。

## 六、核心技术难点与解决方案

### 6\.1 双角色权限隔离（管理员 / 普通用户）

难点：普通用户不可访问后台，管理员不能越权操作用户购物订单。
方案：登录 JWT 令牌携带角色标识，后端拦截器校验接口权限；前端根据角色隐藏对应菜单按钮，无权限接口直接拦截返回提示。

### 6\.2 下单库存数据并发一致性

难点：多用户同时抢购同一款宠物，易出现超卖、库存负数。
方案：下单业务添加`@Transactional`事务注解，扣减库存、生成订单同步执行，异常自动回滚；新增库存锁机制控制并发请求。

### 6\.3 多图批量上传（宠物详情多图、公告封面、头像）

难点：多张图片上传易重名覆盖、前台图片加载失效。
方案：Hutool 工具生成唯一随机文件名，统一静态资源存储目录，Nginx 配置图片访问映射，支持多图批量上传、实时预览。

### 6\.4 多条件复合分页查询

难点：后台宠物、订单、用户数据量大，需要名称、分类、价格多维度筛选。
方案：基于 MyBatis\-Plus 动态拼接查询条件，集成分页插件，高效实现模糊检索、分页加载，适配后台大数据运维场景。

### 6\.5 ECharts 可视化图表数据渲染

难点：后台各类统计图表需要实时查询数据库、动态渲染。
方案：后端封装统计接口，聚合用户、分类、库存数据返回给前端；前端 ECharts 接收 JSON 数据自动渲染饼图、柱状图、折线图。

### 6\.6 全局统一异常处理

难点：库存不足、参数为空、未登录、商品下架等业务异常提示杂乱。
方案：使用`@RestControllerAdvice`全局捕获所有异常，区分业务异常与系统异常，返回标准化友好提示，前端弹窗展示。

### 6\.7 前后端跨域问题

难点：前后端分离端口不同，浏览器同源策略拦截接口请求。
方案：后端全局配置 CORS 跨域规则，允许前端域名、Token 请求头跨域访问，适配开发与线上部署环境。

1. 集成微信 / 支付宝真实支付接口，替换模拟支付，实现完整线上交易；

2. 新增宠物直播、线上预约到店看宠功能；

5. 新增短信通知，下单、发货、论坛回复推送短信提醒用户；

6. 增加宠物医疗、洗护服务板块，拓展平台业务品类；

7. 新增会员积分体系，消费积分可抵扣购宠金额；

8. 定时任务自动下架长期零库存宠物、归档过期公告。

## 九、项目总结

本宠物商城平台基于 SpringBoot\+Vue 前后端分离架构开发，面向线上宠物交易场景，打通宠物展示、选购、支付、资讯、社区交流完整业务链路。系统分为消费者前台商城与管理员运维后台双端，角色权限划分清晰，覆盖宠物分类、宠物商品、购物车订单、资讯公告、论坛互动、数据可视化全维度功能，解决传统线下宠物店选购不便、交易流程不透明、缺少线上交流渠道的痛点。

后端依托 SpringBoot、MyBatis\-Plus 简化数据库开发，JWT 实现安全登录鉴权，事务机制保障库存、订单数据准确；前端采用 Vue3\+Element Plus 搭建简约清新的商城页面与轻量化管理后台，搭配 ECharts 实现直观的数据统计可视化，交互流畅、操作简单。项目涵盖分页检索、多图上传、并发事务、权限控制、图表可视化、全局异常处理、跨域解决等主流企业级开发技术，架构规范、可拓展性强，既满足计算机毕业设计、课程设计开发要求，也可二次开发落地为商用线上宠物商城平台。
